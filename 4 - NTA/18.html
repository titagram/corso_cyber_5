Tcpdump - Filtri

Filtraggio Avanzato del Traffico
Filtri Base

host - Filtra traffico che coinvolge l'host designato (bidirezionale)

src/dest - Modificatori per designare host o porta sorgente/destinazione

net - Mostra traffico da/verso la rete designata (usa notazione /)

proto - Filtra per tipo di protocollo (tcp, udp, icmp)

Filtri Porte

port - Bidirezionale, mostra traffico con la porta specificata come sorgente o destinazione

portrange - Permette di specificare un range di porte (es. 0-1024)

Nota: I filtri porta funzionano meglio quando combinati con il protocollo (es. tcp port 80 per HTTP)

Filtri Dimensione

less/greater - Cerca pacchetti o opzioni di protocollo di una dimensione specifica

Esempio: less 64 mostra solo pacchetti piccoli (spesso SYN, FIN, KeepAlive)

Esempio: greater 500 mostra solo pacchetti grandi (utile per trasferimenti file)

Operatori Logici

and/&& - Concatena due filtri diversi (entrambe le condizioni devono essere soddisfatte)

or - Match su una delle due condizioni (non devono essere entrambe soddisfatte)

not/! - Modificatore che dice "tutto tranne x" (es. not udp)

Esempi Pratici
# Cattura traffico da un host specifico sudo tcpdump -i eth0 host 192.168.1.100 # Solo traffico TCP sulla porta 443 sudo tcpdump -i eth0 tcp port 443 # Traffico da una rete specifica sudo tcpdump -i eth0 dest net 192.168.1.0/24 # Escludi ICMP sudo tcpdump -i eth0 not icmp # Combinazione di filtri sudo tcpdump -i eth0 host 192.168.1.100 and tcp port 80
