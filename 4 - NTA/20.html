Interrogare il Traffico di Rete
Obiettivo del Lab

Fornire esposizione all'interrogazione del traffico di rete e pratica preziosa nell'implementazione di filtri di pacchetti. Utilizzeremo filtri come host, port, protocol e altro per cambiare la nostra vista mentre scaviamo attraverso un file .PCAP.

Scenario: Il management ci ha incaricato di eseguire un'analisi rapida del traffico catturato dal team. L'obiettivo è determinare quali server stanno rispondendo a richieste DNS e HTTP/S nella nostra rete locale.

Task #1: Lettura Base

Leggere una cattura da un file senza filtri implementati. Esaminare il PCAP con nessun filtro applicato per avere una visione generale del traffico.

Task #2: Identificare Tipo di Traffico

Prendere nota dei tipi di traffico visibili (porte utilizzate, protocolli, altre informazioni rilevanti). Quali filtri possiamo usare per facilitare questo compito?

Task #3: Identificare Conversazioni

Determinare se notiamo pattern nel traffico. Ci sono connessioni comuni tra un server e un host? Chi sono i server e i client in queste conversazioni?

Task #4: Analisi Approfondita

Utilizzare qualsiasi sintassi necessaria per rispondere a domande specifiche: timestamp della prima conversazione, indirizzi IP di domini, protocolli utilizzati.

Task #5-7: Filtraggio e Analisi
  • Task #5: Filtrare tutto il traffico che non è DNS. Chi è il server DNS? Quali nomi di dominio sono stati richiesti?
  • Task #6: Filtrare per traffico TCP (HTTP/HTTPS). Quali pagine web sono state richieste? Quali sono i metodi HTTP più comuni?
  • Task #7: Esaminare il webserver nella prima conversazione. Possiamo determinare quale applicazione sta eseguendo il webserver?
