HTTP e HTTPS

Protocolli del Livello Applicativo
HTTP - Hypertext Transfer Protocol

Protocollo stateless del livello Applicativo in uso dal 1990.

Funzionalità: Abilita il trasferimento di dati in chiaro tra client e server su TCP

Porte: 80 o 8000 su TCP (normalmente)

Il client invia una richiesta HTTP al server chiedendo una risorsa. Il server risponde con il media richiesto (HTML, immagini, video).

HTTPS - HTTP Secure

Modifica del protocollo HTTP progettata per utilizzare TLS (Transport Layer Security) o SSL per la sicurezza dei dati.

Porte: 443 e 8443 invece della porta standard 80

Vantaggi: Cifra l'intera conversazione, non solo i dati inviati o richiesti. Protegge da attacchi Man-in-the-Middle.

Metodi HTTP
Metodo Descrizione
GET Richiede informazioni e contenuto dal server (più comune)
POST Invia informazioni al server basate sui campi nella richiesta
PUT Crea o aggiorna un oggetto all'URI fornito
DELETE Rimuove l'oggetto all'URI dato
HEAD Richiede informazioni sul server senza il body del messaggio
