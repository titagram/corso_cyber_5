TCP Three-Way Handshake

Stabilimento di una Sessione TCP
1. SYN (Client → Server)

Il client invia un pacchetto con il flag SYN attivo insieme ad altre opzioni negoziabili nell'header TCP.

Questo è un pacchetto di sincronizzazione che permette a entrambe le estremità di concordare un numero di sequenza per iniziare a comunicare.

Vengono negoziate anche: dimensione della finestra, maximum segment size, selective acknowledgments.

2. SYN-ACK (Server → Client)

Il server risponde con un pacchetto TCP che include:

  • Flag SYN per la negoziazione del numero di sequenza
  • Flag ACK per riconoscere il pacchetto SYN precedente

Il server include anche eventuali modifiche alle opzioni TCP richieste.

3. ACK (Client → Server)

Il client risponde con un pacchetto TCP con il flag ACK attivo, accettando la negoziazione.

Questo pacchetto conclude il three-way handshake e stabilisce la connessione tra client e server.

Session Teardown

Per terminare una connessione TCP, viene utilizzato il flag FIN.

Pattern:

  • FIN, ACK
  • FIN, ACK
  • ACK

Questo pattern indica una connessione adeguatamente terminata.
