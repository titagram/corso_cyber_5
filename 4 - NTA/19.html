Tcpdump - Flag TCP e Output

Analisi Avanzata dei Pacchetti
Cercare Flag TCP

Per vedere solo pacchetti con il flag SYN impostato:

tcpdump -i eth0 'tcp[13] & 2 != 0'

Questo conta fino al 13° byte nella struttura e guarda il 2° bit. Se è impostato a 1 o ON, il flag SYN è impostato.

Opzioni Output

-S - Mostra numeri di sequenza assoluti invece di relativi

-v, -vv, -vvv - Aumenta la verbosità dell'output

-A - Mostra solo testo ASCII dopo la riga del pacchetto

-l - Output line-buffered (utile per piping a grep)

Interpretazione Output Tcpdump

L'output di Tcpdump può essere suddiviso in diversi campi:

  • Timestamp: Data e ora del pacchetto
  • Protocol: Protocollo del livello superiore (IP, TCP, UDP, ecc.)
  • Source & Destination IP.Port: IP e porta sorgente/destinazione
  • Flags: Flag TCP utilizzati (SYN, ACK, FIN, ecc.)
  • Sequence e Acknowledgement Numbers: Numeri usati per tracciare i segmenti TCP
  • Protocol Options: Valori TCP negoziati (window size, selective acknowledgments, ecc.)
Pre-Capture vs Post-Capture Filtering

Pre-Capture: Applicare filtri direttamente alla cattura farà cadere qualsiasi traffico non corrispondente. Riduce la quantità di dati nelle catture ma può eliminare traffico potenzialmente necessario in seguito.

Post-Capture: Applicare filtri quando si legge un file di cattura parserà il file e rimuoverà qualsiasi cosa non corrispondente dall'output del terminale. Non modifica permanentemente il file di cattura.
