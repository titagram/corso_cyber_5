API Flow: ReadProcessMemory

Anatomia di una chiamata

Immagina un malware che vuole leggere la memoria di lsass.exe per rubare password.

  1. Il malware chiama ReadProcessMemory (in Kernel32.dll).
  2. Kernel32 chiama NtReadVirtualMemory (in Ntdll.dll).
  3. Ntdll esegue l'istruzione SYSCALL.
  4. La CPU passa in Kernel Mode.
  5. Il Kernel esegue l'azione e restituisce i dati.
