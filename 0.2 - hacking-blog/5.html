La Logica del Bug: La funzione di ritaglio immagini (`wp_crop_image`) in questa versione non validava correttamente l'estensione del file di output.

L'Attacco:

  1. Carichiamo un'immagine legittima contenente codice PHP iniettato (exif data o append).
  2. Chiediamo al server di "ritagliarla".
  3. Manipoliamo la richiesta per salvare il ritaglio come `.php` invece di `.jpg`.
  4. Il server esegue il codice PHP quando visitiamo l'URL dell'immagine.
