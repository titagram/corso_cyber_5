SPL: Comandi Base

fields, table, rename, dedup, sort
fields

Specifica quali campi includere o escludere nei risultati della ricerca.

Esempio:

index="main" sourcetype="WinEventLog:Sysmon" EventCode=1 | fields - User

Dopo aver recuperato tutti gli eventi di creazione processo dall'indice main, il comando fields esclude il campo User dai risultati. Utilizzando sourcetype si restringe l'ambito esclusivamente ai log Sysmon.

table

Presenta i risultati della ricerca in formato tabellare.

Esempio:

index="main" sourcetype="WinEventLog:Sysmon" EventCode=1 | table _time, host, Image

Questa query restituisce eventi di creazione processo, poi organizza i campi selezionati (_time, host, e Image) in formato tabellare. _time è il timestamp dell'evento, host è il nome dell'host, e Image è il nome del file eseguibile.

rename

Rinomina un campo nei risultati della ricerca.

Esempio:

index="main" sourcetype="WinEventLog:Sysmon" EventCode=1 | rename Image as Process

Questo comando rinomina il campo Image in Process nei risultati. Il campo Image nei log Sysmon rappresenta il nome del file eseguibile del processo.

dedup

Rimuove eventi duplicati.

Esempio:

index="main" sourcetype="WinEventLog:Sysmon" EventCode=1 | dedup Image

Il comando dedup rimuove voci duplicate basate sul campo Image dagli eventi di creazione processo. Se lo stesso processo viene creato più volte, apparirà solo una volta nei risultati.

sort

Ordina i risultati della ricerca.

Esempio:

index="main" sourcetype="WinEventLog:Sysmon" EventCode=1 | sort - _time

Questo comando ordina tutti gli eventi di creazione processo nell'indice main in ordine decrescente dei loro timestamp (_time), cioè gli eventi più recenti vengono mostrati per primi. Il segno - indica ordine decrescente.
