SPL: Lookup e Inputlookup

Arricchimento Dati con Fonti Esterne
lookup

Arricchisce i dati con fonti esterne (es. tabelle CSV).

Esempio di CSV:

filename, is_malware notepad.exe, false cmd.exe, false powershell.exe, false sharphound.exe, true randomfile.exe, true

Questo file CSV deve essere aggiunto come nuova tabella Lookup in Splunk.

Query con lookup

Query completa:

index="main" sourcetype="WinEventLog:Sysmon" EventCode=1 | rex field=Image "(?P<filename>[^\\\]+)$" | eval filename=lower(filename) | lookup malware_lookup.csv filename OUTPUTNEW is_malware | table filename, is_malware
Spiegazione della Query

1. Ricerca: Cerca eventi di creazione processo (EventCode=1) nei log Sysmon.

2. rex: Estrae il nome del file dal campo Image (tutto dopo l'ultimo backslash).

3. eval: Converte il nome del file in minuscolo per ricerca case-insensitive.

4. lookup: Confronta il nome del file con la tabella lookup. Se trova una corrispondenza, aggiunge il campo is_malware all'evento.

5. table: Mostra solo i campi filename e is_malware. Se is_malware non è presente in una riga, significa che non è stata trovata corrispondenza nella tabella lookup.

inputlookup

Recupera dati da un file lookup senza unirli ai risultati della ricerca.

Esempio:

| inputlookup malware_lookup.csv

Questo comando recupera tutti i record dal file malware_lookup.csv. Il risultato non viene unito con alcun risultato di ricerca, ma può essere usato per verificare il contenuto del file lookup o per operazioni successive come filtraggio o join con altri dataset.
