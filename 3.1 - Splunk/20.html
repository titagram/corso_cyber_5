Identificare Anomalie nei Dati
Identificare EventCodes Sysmon

Applichiamo lo stesso approccio e identifichiamo tutti gli EventCodes Sysmon prevalenti nei nostri dati:

index="main" sourcetype="WinEventLog:Sysmon" | stats count by EventCode

La scansione scopre 20 EventCodes distinti.

EventCodes Chiave per Detection

Event ID 1 - Process Creation: Utile per hunt che mirano a gerarchie parent-child anomale.

Event ID 3 - Network connection: Fonte di rumore abbondante, ma può rivelare anomalie.

Event ID 10 - ProcessAccess: Utile per individuare iniezione di codice remota e memory dumping.

Event ID 11 - FileCreate: Può essere utile per correlare o identificare origini di file.

Event ID 13 - RegistryEvent: Molti eventi registry possono essere malevoli.

Event ID 17/18 - Pipe created/connected: Aiutano a osservare comunicazione interprocesso di malware, uso di PsExec e movimento laterale SMB.

Event ID 22 - DNSEvent: Traccia query DNS, utile per monitorare risoluzioni beacon e DNS beacons.

Query Preliminari

Basandoci su questi EventCodes, possiamo eseguire query preliminari. Come affermato in precedenza, alberi parent-child insoliti sono sempre sospetti.

Query per ispezionare tutti gli alberi parent-child:

index="main" sourcetype="WinEventLog:Sysmon" EventCode=1 | stats count by ParentImage, Image

Ci troviamo di fronte a 5.427 eventi, un bel mucchio da setacciare manualmente. Abbiamo scelte: eliminare ciò che sembra benigno o mirare a processi child noti per essere problematici, come cmd.exe o powershell.exe.
