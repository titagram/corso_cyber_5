Intrusion Detection con Splunk

Scenario Reale
Contesto

Passiamo dall'esplorazione di log su una singola macchina all'investigazione su scala molto più ampia, attraverso numerose macchine per scoprire attività irregolari nell'intera rete.

Strumenti: Ancora Windows Event logs, ma con uno scope significativamente più ampio.

Obiettivi

In questo modulo:

  • Ci concentreremo su macchine specifiche malevole
  • Padroneggeremo l'arte di creare query precise
  • Attiveremo alert per migliorare proattivamente la sicurezza dell'ambiente
  • Identificheremo e scarteremo falsi positivi
Strategia

La strategia che seguiremo per identificare eventi rispecchierà le lezioni iniziali. Tuttavia, ora ci confrontiamo con un dataset molto più grande invece di una singola collezione di log eventi.

Dal punto di vista della dashboard Splunk, mireremo a eliminare i falsi positivi.

Ingesting Data Sources

Per procedere, abbiamo bisogno di accesso a dati che possiamo analizzare per threat hunting.

Fonti disponibili:

  • BOTS: Fornito da Splunk con istruzioni di installazione
  • nginx_json_logs: Risorsa utile che fornisce log dummy in formato JSON
  • Dataset personalizzato: Oltre 500.000 eventi con multiple infezioni

Query per recuperare tutti gli eventi:

index="main" earliest=0
