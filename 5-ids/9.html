Suricata - Aggiornamento Regole

Live Reload e suricata-update
Live Rule Reloading

Funzionalità cruciale che permette di aggiornare il nostro ruleset senza interrompere l'ispezione del traffico in corso.

Per abilitare, configurare in suricata.yaml:

detect-engine: - reload: true

Poi eseguire:

sudo kill -usr2 $(pidof suricata)
suricata-update

Strumento per aggiornare il ruleset di Suricata:

sudo suricata-update

Questo comando recupera le regole stabilendo una connessione con https://rules.emergingthreats.net/open/ e salva le regole appena ottenute nella directory /var/lib/suricata/rules/.

Lista Fonti di Regole

Per generare una lista completa di tutti i provider di ruleset:

sudo suricata-update list-sources

Fonti comuni includono:

  • et/open - Emerging Threats Open Ruleset
  • et/pro - Emerging Threats Pro (commerciale)
  • oisf/trafficid - Suricata Traffic ID ruleset
Abilitare una Fonte

Per abilitare una fonte specifica (es. et/open):

sudo suricata-update enable-source et/open

Poi rieseguire suricata-update per caricare il ruleset appena acquisito. Potrebbe essere richiesto un riavvio del servizio Suricata.

Validazione Configurazione

Per validare la configurazione di Suricata:

sudo suricata -T -c /etc/suricata/suricata.yaml

Questo comando esegue un test per verificare se il file di configurazione è valido e tutti i file referenziati nella configurazione sono accessibili.
