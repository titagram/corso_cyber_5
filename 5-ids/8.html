Suricata - File Extraction

Estrazione File dal Traffico
Abilitare File Extraction

Suricata ha una funzionalità sottoutilizzata ma altamente potente: file extraction. Questa funzionalità ci permette di catturare e memorizzare file trasferiti su diversi protocolli.

Per abilitare l'estrazione file, modifichiamo la sezione file-store in suricata.yaml:

file-store: version: 2 enabled: yes force-filestore: yes dir: /path/to/filestore
Regola per File Extraction

L'estrazione file non è un processo automatico. Dobbiamo creare una regola specifica che istruisca Suricata quando e che tipo di file estrarre.

Esempio di regola semplice:

alert http any any -> any any (msg:"FILE store all"; filestore; sid:2; rev:1;)
Organizzazione File Store

Il modulo file-store utilizza la sua directory di log (default: filestore nella directory di logging di default) e registra file usando lo SHA256 del contenuto come nome del file.

Ogni file è poi posizionato in una directory nominata da 00 a ff dove la directory condivide i primi 2 caratteri del nome del file.

Esempio: Se lo SHA256 inizia con f9bc6d..., il file sarà posizionato in filestore/f9/
