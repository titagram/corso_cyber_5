Architettura e Log di Zeek

Componenti e Output
Architettura di Zeek

L'architettura di Zeek comprende due componenti primarie:

1. Event Engine (Core): Prende un flusso di pacchetti in entrata e lo trasforma in una serie di eventi di alto livello. Questi eventi descrivono attività di rete in termini policy-neutral, informandoci di cosa sta succedendo senza offrire interpretazione.

2. Script Interpreter: Esegue un insieme di event handler scritti nel linguaggio di scripting Zeek, esprimendo la policy di sicurezza del sito.

Processo di Eventi

Gli eventi generati dal core di Zeek sono accodati in modo ordinato, aspettando il loro turno per essere processati su base first-come-first-served.

La maggior parte degli eventi di Zeek sono definiti in file .bif localizzati in /scripts/base/bif/plugins/.

L'interpretazione o analisi è fornita dallo script interpreter, che definisce azioni da intraprendere al rilevamento di certi eventi.

Log Principali di Zeek
Log Descrizione
conn.log Dettagli su connessioni IP, TCP, UDP e ICMP
dns.log Dettagli di query e risposte DNS
http.log Dettagli di richieste e risposte HTTP
ftp.log Dettagli di richieste e risposte FTP
smtp.log Transazioni SMTP, come dettagli mittente e destinatario
Campi http.log

Il file http.log è pieno di dati preziosi come:

  • host: Il dominio/IP HTTP
  • uri: L'URI HTTP
  • referrer: Il referrer della richiesta HTTP
  • user_agent: L'user agent del client
  • status_code: Il codice di stato HTTP
