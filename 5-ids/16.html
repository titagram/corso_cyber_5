Esempio Regola: Sliver (TLS)

Rilevamento Basato su JA3 Hash
Regola per Sliver C2 SSL
alert tls any any -> any any (msg:"Sliver C2 SSL"; ja3.hash; content:"473cd7cb9faa642487833865d516e578"; sid:1002; rev:1;)

Questa regola Suricata è progettata per rilevare certe varianti di Sliver (un altro framework C2 open-source) ogni volta che identifica una connessione TLS con un hash JA3 specifico.

Come Funziona

ja3.hash: Utilizza il buffer JA3 per match sull'hash JA3

content: Cerca l'hash JA3 specifico 473cd7cb9faa642487833865d516e578

Questo hash è unico per il client Sliver e può essere calcolato dal traffico di rete utilizzando strumenti come ja3.

Vantaggi del JA3

Il fingerprinting JA3 è efficace perché:

  • È basato su caratteristiche del client hello che rimangono visibili anche in traffico cifrato
  • Può identificare specifiche versioni o configurazioni di malware
  • Non richiede decifratura del traffico
  • È difficile da modificare senza cambiare il comportamento del client
Calcolo JA3 Hash

L'hash JA3 può essere calcolato da un file PCAP utilizzando strumenti come:

ja3 -a --json /path/to/file.pcap

Questo strumento analizza i pacchetti Client Hello e genera l'hash JA3 unico per ogni connessione TLS, permettendoci di identificare pattern specifici associati a malware o software sospetti.
