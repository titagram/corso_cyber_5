Zeek - Caratteristiche Chiave

Funzionalità Avanzate
Caratteristiche Principali
  • Logging completo: Registrazione completa delle attività di rete
  • Analisi protocolli application-layer: Indipendentemente dalla porta, copre protocolli come HTTP, DNS, FTP, SMTP, SSH, SSL, ecc.
  • Ispezione contenuto file: Capacità di ispezionare contenuto di file scambiati su protocolli application-layer
  • Supporto IPv6: Supporto completo per IPv6
  • Rilevamento e analisi tunnel: Rilevamento e analisi di tunnel
Capacità Aggiuntive
  • Capacità di condurre sanity check durante l'analisi del protocollo
  • Pattern matching simile a IDS
  • Linguaggio di scripting potente e domain-aware
  • Interfacciamento che output a log ASCII ben strutturati di default
Integrazione e Estendibilità
  • Backend alternativi per ElasticSearch e DataSeries
  • Integrazione real-time di input esterni nelle analisi
  • Libreria C esterna per condividere eventi Zeek con programmi esterni
  • Capacità di attivare processi esterni arbitrari dal linguaggio di scripting
Utilizzo Base di Zeek

Per analisi offline di un file PCAP:

/usr/local/zeek/bin/zeek -C -r /path/to/file.pcap

I log saranno memorizzati nella directory corrente. Zeek, nella sua configurazione standard, applica compressione gzip ai file di log ogni ora. I log più vecchi sono poi trasferiti in una directory nominata nel formato YYYY-MM-DD.

Strumenti utili: gzcat per stampare log compressi, zgrep per cercare all'interno di log compressi, e zeek-cut per gestire file di log Zeek.
