Configurazione Suricata

File di Regole e Variabili
File di Regole

Suricata carica regole da file nella directory /etc/suricata/rules/. Possiamo visualizzare tutti i file di regole con:

ls -lah /etc/suricata/rules/

Le regole possono essere commentate (non caricate) quando una nuova versione entra in gioco o se la minaccia associata diventa obsoleta o irrilevante.

Variabili nelle Regole

Ogni regola di solito coinvolge variabili specifiche come:

  • $HOME_NET - Rete interna
  • $EXTERNAL_NET - Rete esterna

La regola esamina traffico dagli indirizzi IP specificati in $HOME_NET verso gli indirizzi IP in $EXTERNAL_NET.

Configurazione suricata.yaml

Queste variabili possono essere definite nel file di configurazione suricata.yaml:

vars: address-groups: HOME_NET: "[10.0.0.0/8]" EXTERNAL_NET: "!$HOME_NET"

Questo ci permette di personalizzare queste variabili secondo il nostro ambiente di rete specifico e persino definire le nostre variabili.

Caricare Regole Personalizzate

Per configurare Suricata per caricare signature da un file di regole personalizzato (come local.rules), dobbiamo aggiungere il percorso al file suricata.yaml nella sezione rule-files:.

Questo permette di testare e sviluppare regole personalizzate senza modificare i file di regole di sistema.
