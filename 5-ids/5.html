Suricata - Input e Output

Modalità di Cattura e Logging
Offline Input

Legge file PCAP per processare pacchetti precedentemente catturati nel formato file LibPCAP.

Vantaggi:

  • Utile per esami post-mortem dei dati
  • Strumentale quando si sperimentano vari set di regole e configurazioni
Live Input

LibPCAP: Pacchetti letti direttamente da interfacce di rete. Limitato da limitazioni di performance e mancanza di capacità di load-balancing.

NFQ: Modalità inline IPS specifica per Linux, collabora con IPTables per deviare pacchetti dallo spazio kernel in Suricata.

AF_PACKET: Miglioramento delle performance rispetto a LibPCAP e supporta multi-threading. Non compatibile con distribuzioni Linux più vecchie.

Output di Suricata

EVE (JSON): Output raccomandato di Suricata. File JSON formattato che registra un'ampia gamma di tipi di eventi inclusi alert, HTTP, DNS, metadati TLS, drop, metadati SMTP, flow, netflow e altro. Facilmente consumabile da strumenti come Logstash.

Unified2: Formato binario di alert di Snort, abilitando l'integrazione con altro software che utilizza Unified2. Può essere letto usando lo strumento u2spewfoo di Snort.

fast.log: Formato log basato su testo che registra solo alert ed è abilitato di default.

stats.log: Log di statistiche leggibile dall'uomo, particolarmente utile durante il debug di deployment di Suricata.
